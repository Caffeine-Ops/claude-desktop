确认页底栏 · 已选摘要 Tag 方案

底栏白底已定稿，本页只探索「已选摘要」这排 tag 的形态——从最保守（现状精修）到最大胆（去胶囊化）共 6 个方向，色值全部取自 design-tokens，可直接对照落码。

tokens: bg #F7F7F9 · card #FFF · muted #ECECF0 · border #D3D3DB · brand #16A34A
01 描边胶囊 · 现状精修

当前线上形态的微调：白底胶囊 + 细描边。最安静、信息感最弱，白色底栏上仅靠描边勾勒轮廓，适合不想让摘要抢视线的取向。

fill #FFF · border #D3D3DB · text #62626A · radius 999 · padding 6.5×12

02 灰底填充 推荐

无描边、浅灰实底——GitHub topics / Linear labels 的经典 tag 语言。白色底栏上区分度天然充足，不依赖描边或阴影撑轮廓，深浅关系也不受用户自定义主题影响，是最不容易再出「看起来发灰」问题的形态。

fill #ECECF0 (muted) · no border · text #46464D · radius 999 · padding 6.5×12
横幅 简报 像素风
03 轨道分组

整排 tag 收进一条浅灰轨道，白色胶囊浮在轨道上（segmented control 的「凹槽+凸起」语言），轨道左端带「已选」小字标明语义。分组感最强，适合底栏元素继续变多之后仍要保持秩序的场景。

track #F1F0F4 p4 · chip #FFF shadow 0 1 2 /10% · label 11px #8E8E96
已选 横幅 简报 像素风
04 键值对胶囊

每颗胶囊里带上维度名（画布 / 用途 / 风格），信息量最高——用户不用记「第二颗是什么维度」，扫一眼就知道每个值对应什么。代价是每颗胶囊更宽，选项多时更容易触发横向滚动。

fill #F4F3F7 · dim-label 10.5px #8E8E96 · value 12px #46464D · gap 7

05 品牌绿点缀

胶囊用品牌绿 9% 淡底 + 绿字 + 勾选图标，与右侧确认按钮形成同色呼应——「这些是你确认过的选择」的语义最强，整条底栏读作一个绿色系的确认动线。适合想强化「已确认」感的取向；若嫌绿色太多可去掉勾。

fill brand/9% · text #15803D · check 10px stroke 2.5 · radius 999
横幅 简报 像素风
06 极简去胶囊

反方向的大胆解：彻底去掉胶囊，回到文字但重建层级——「已选」小字 + 分隔线定语义，值用 13px 中字重深灰，间隔点退到最浅。底栏视觉最轻、最不打扰，把全部注意力让给内容区和确认按钮。

label 11px #8E8E96 + 1px divider · value 13px/500 #46464D · dot #C2C1C9
