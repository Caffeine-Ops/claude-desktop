# POC / 试点方案 · 通用骨架〔通用款·待真实样本升级〕

> ⚠️ **通用款**：本卡按行业通用套路搭，**尚无真实样本提炼**。有了真实 POC/试点样本请据此升级本卡。

**类型定位**：正式落地前的小范围验证方案，核心是**讲清验证什么、怎么验、验成什么样算成功**。
篇幅通常比建设方案短，重"目标—指标—排期—评判"闭环。

## 章节顺序 + 调用的 section 卡

- **封面** → `sections/cover.md`
- **一、POC 背景与目标**
  - 背景 / 要验证的假设 → `sections/overview/background.md`
  - 试点目标（明确、可衡量）→ `sections/overview/goals-by-audience.md` 或简化版
- **二、试点范围**
  - 范围边界（做什么、不做什么）→ `sections/overview/business-scope.md`
  - 试点环境 / 数据 / 参与方
- **三、验证方案**
  - 要验证的能力清单 → `sections/features/`（挑核心几项，不铺全量）
  - **验证指标与评判标准**（关键）：指标 → 目标值 → 测法，用表格 `methodology/tables.md`
- **四、实施排期**
  - 试点时间表 / 里程碑（甘特图）→ `sections/delivery/implementation-plan.md`
  - 阶段划分（准备 → 部署 → 试运行 → 评估）
- **五、成功标准与后续**
  - 通过/不通过的判定条件
  - 试点成功后的推广路径

## 行业适配提示（内联）

- 医疗：试点常选 1–2 个科室，指标如引用率、门诊转化、医生满意度。
- 金融/政务：重合规验证、数据安全验证。
- 通用：重 ROI 验证、可行性验证。

## 写法要点

- 核心是**闭环**：目标 → 指标 → 排期 → 评判，四者对齐，别只讲功能不讲怎么算成功。
- 指标目标值、排期——只能来自素材/用户，编不得；无据标 `⚠️ 资料缺失`。
- 篇幅偏精简（见 `emphasis-and-depth.md`），别写成完整建设方案。
