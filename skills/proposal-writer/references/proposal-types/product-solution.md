# 产品方案 · 通用骨架〔通用款·待真实样本升级〕

> ⚠️ **通用款**：本卡按行业通用套路搭，**尚无真实样本提炼**。有了真实产品方案样本请据此升级本卡。

**类型定位**：偏产品能力/价值叙事，给客户业务侧/决策层看——讲清产品是什么、能力亮点、用在什么场景、带来什么收益。
比技术方案更重"价值与体验"，比售前建设方案更聚焦"产品本身"。

## 章节顺序 + 调用的 section 卡

- **封面** → `sections/cover.md`
- **一、产品概述**
  - 产品定位 → `sections/overview/positioning.md`
  - 产品总体概述 → `sections/overview/product-summary.md`
  - 目标用户 / 适用场景 → `sections/overview/business-scope.md`
- **二、核心能力 / 功能亮点**（本方案重心，重点详写）
  - 按功能域用 `sections/features/` 各形态卡，突出"亮点 + 用户价值"
- **三、应用场景**
  - 典型场景走查（场景 + 该场景下产品怎么用 + 效果），可逐条 → `presentation-modes.md`
- **四、客户价值 / 收益**
  - 分价值点 → `sections/overview/value.md`
  - 成效 / 案例数据 → `sections/overview/outcomes.md`（数据必须有素材出处）
- **五、产品架构**（轻量）→ `sections/architecture/overall-architecture.md`

## 行业适配提示（内联，无样本时的常识，不替代素材）

- 零售/电商：强调转化、复购、用户运营。
- 医疗：强调就医体验、效率、合规评级。
- 教育：强调教学效果、减负、个性化。
- 通用企业：强调降本增效、易用、可集成。

## 写法要点

- 产品方案可比技术方案多一点营销叙事，但**收益/数据仍不许编**——"提升 30%"这类必须有出处，否则标缺料。
- 亮点功能配产品截图（满血）或占位（独立）——图文并茂尤其重要。
- 受众是业务/决策层，语言更通俗、重价值而非实现细节。
