# 投标 / 商务方案 · 通用骨架〔通用款·待真实样本升级〕

> ⚠️ **通用款**：本卡按行业通用套路搭，**尚无真实样本提炼**。有了真实投标/商务样本请据此升级本卡。
> 特别提醒：投标方案强依赖招标文件的"应答点"，**格式合规**优先——真实招标要求只能来自素材。

**类型定位**：面向招投标/采购场景，核心是**逐条应答招标需求 + 展示资质与商务条件**。
格式合规（章节对应评分项）比文采更重要。

## 章节顺序 + 调用的 section 卡

- **封面**（投标方案封面，含项目名、投标单位）→ `sections/cover.md`
- **一、公司资质与实力**：资质证书、类似案例（数据/证书号必须来自素材，不编）
- **二、需求理解与响应**
  - 对招标需求的理解 → `sections/overview/background.md`
  - **需求点逐条应答**：招标要求 → 我方响应，用表格 → `sections/features/`+`tables.md`
- **三、技术方案**（按招标要求繁简）→ 复用 `architecture/` + `features/`
- **四、实施方案**
  - 实施计划 / 里程碑 → `sections/delivery/implementation-plan.md`
  - 项目组织 / 人员配备（视素材）
- **五、服务与承诺**
  - 售后服务承诺 → `sections/delivery/after-sales.md`
  - 培训、质保等承诺
- **六、商务报价** → `sections/delivery/pricing-config.md`（报价项必须来自素材，缺写「—」不编）

## 行业适配提示（内联）

- 政务/国企：极重资质、信创、合规条款、盖章件——按招标文件清单逐项对应。
- 大型企业采购：重 SLA、服务响应时效、罚则响应。

## 写法要点

- **应答表**是核心：三列「招标要求 | 我方响应 | 满足程度」，逐条对应，别漏项（漏项＝丢分）。
- 资质、案例、报价、人员——全是**易编造重灾区**，一律只填素材有的，无据标 `⚠️ 资料缺失`。
- 若用户给了招标文件，先解析其评分项/应答点，据此定章节；这是本类型最关键的一步。
