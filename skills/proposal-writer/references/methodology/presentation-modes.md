# 呈现方式（逐条 / 段落 / 表格 / 图文）

同样的事实可以有不同呈现。按内容形态 + 用户偏好（见 `emphasis-and-depth.md` 与第 0.5 步采集）选。

## 四种呈现，各自何时用

- **逐条列举**：功能点、卖点、价值点、清单类——每条「小标题 + 该条内容 + 来源」，不要并成一段。
  用户说"一条条介绍""逐条"的部分**必须**逐条。
- **散文段落**：背景、定位、理念、综述类——需要连贯论述、承上启下的内容。
- **表格**：结构化数据（参数/对比/清单/计划/报价）——见 `tables.md`。
- **图文并茂**：功能详述、架构、统计分析——正文 + 截图/结构图配合，见 `images-and-figures.md`。

## 怎么写好每种

- 逐条：小标题用**简短名词短语**（如"多轮对话""科室推荐"），每条 1–3 句说清"是什么 + 有什么用"，末尾标来源。
- 段落：一段一个意思，别把五件事塞一段；空行分段，不用 HTML 换行。
- 表格：表头用源料字段名，缺值写「—」。
- 图文：一节一图为宜，图说一句话点明图意。

## 尊重用户随时下的指令

用户中途说"这节改成表格""这段太长，逐条"——立即照办，不必反问。默认由你按形态判断，拿不准才问。

## 正例（逐条）

> **多轮对话**
> 支持连续追问，逐步收敛症状描述，直至给出科室建议。（据《智能导诊产品说明》）
>
> **科室及医生推荐**
> 依据对话结果推荐匹配科室与出诊医生，可一键跳转挂号。（据《智能导诊产品说明》）
