# 独立流程：一口气 vs 逐段确认

独立模式（app 外）的 generate 流程有两档，**默认一口气**，可切逐段。

## 默认：一口气出初稿

1. 判类型 → 读骨架卡 → 定章节大纲。
2. 用素材**逐章连续产出**：封面 → 目录 → 正文各章（逐章写，见 `SKILL.md` 长文档一致性）。
3. 全篇成稿后**一次性**征求修改意见。
4. 全程不强制每阶段停下等确认；缺料就地标注、不打断。

## 可选：逐段确认（用户说"我们一步步来 / 分封面·目录·正文逐段确认"时切换）

按三阶段有序推进，每阶段停下等用户放行：

1. **封面**：先用 AskUserQuestion 问齐关键信息（客户单位全称、方案标题、落款单位与日期），再写封面。
   写完发起"封面确认"，用户点放行才进下一阶段。
2. **目录**：给章节大纲（嵌套有序列表、层级适度、只写标题不写序号）。发起"目录确认"，放行才写正文。
3. **正文**：严格按已确认目录逐章写，不自行增删章节。

## 两模式默认相反（别混淆）

- **满血模式**默认逐段确认（正式交付、稳字优先，app 强制）。
- **独立模式**默认一口气（轻量试写、快字优先）。

同一个人用两边会看到不同默认，这是有意为之——若用户困惑，一句话解释即可。

## 注意

- 逐段确认时，"确认卡"本身属于提问，不算方案正文。
- 独立模式没有 app 的哨兵/硬门，逐段确认靠你自觉停下等放行，别抢跑。
