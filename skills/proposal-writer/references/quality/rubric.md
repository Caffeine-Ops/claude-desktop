# 交付前质量自检清单

写完（或改完）方案，交付前逐条自检。任一项不过关就回去修。

## 接地（最高优先）

- [ ] 正文每段末尾都有 `（据《文件名》）` 来源标注？（封面/目录除外）
- [ ] 没有任何编造——数据、功能、案例、报价、排期都能在素材里找到出处？
- [ ] 查不到的地方都用「⚠️ 资料缺失：<具体>」标了，而不是编或跳过？
- [ ] 独立模式：没有编造的图片路径？（截图处是占位「〔建议插入…〕」或真实路径）

## 结构

- [ ] 章节顺序、层级符合类型骨架（`proposal-types/`）？
- [ ] 正文标题层级与目录一一对应，没把深层小节压平？
- [ ] 封面结构正确（标题/抬头/`---`/落款），没插图、没标来源、没写 HTML？
- [ ] 目录只写标题、没自写序号（满血由导出器编号；独立模式手写序号）？

## 图文并茂

- [ ] 叶子功能小节该配截图的配了（满血）/ 写了占位（独立）？
- [ ] 每个一级章有事实可依的配了结构图（mermaid 或门面图）？
- [ ] mermaid 含中文标点的节点标签用英文双引号包裹了？没画不出的图？
- [ ] 结构化数据（参数/对比/清单/计划/报价）用了表格、缺值写「—」？

## 呈现与偏好

- [ ] 用户要"逐条"的地方逐条了、要重点的章节展开了？
- [ ] 呈现方式贴合内容形态与用户偏好（`methodology/presentation-modes.md`）？
- [ ] 篇幅/受众语气符合用户要求？

## 干净

- [ ] 正文只有纯 markdown，没有任何 HTML 标签？
- [ ] 提问/确认/思路说明没混进方案正文？

## 改写任务额外自检

- [ ] 只改了用户要动的部分，其余原样保留（最小改动）？
- [ ] 新增内容有素材依据，无据的标了缺料？
