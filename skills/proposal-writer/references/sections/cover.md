# 封面

方案第一页，只含标题 + 抬头 + 落款，独占一页。**不插图、不标来源、不加"封面"二字。**

## 缺失信息优先提问顺序（写封面前问齐）

1. 客户单位全称
2. 方案主题 / 标题
3. 编制（落款）单位与日期

（这几项合并进一次 AskUserQuestion 问；开放式信息给候选猜测，用户用"其他"补。）

## 结构骨架（逐行）

```markdown
# <方案标题>

<客户单位>
<副标题 / 项目名（可选）>

---

<编制单位>
<日期>
```

- 标题用一级标题 `#` 开头。
- 其下写客户单位等抬头信息，每项一行。
- 单独一行写一个 `---`（三个连字符）作分隔。
- `---` 之下写编制单位与日期等落款，每项一行。

## 两种模式差异

- **满血模式**：导出器据这个结构把标题排在页面上中部、落款贴底部、整页独占——你**只写纯 markdown**，
  不要加任何居中/分页标签、不要写序号。
- **独立模式**：无导出器，就按上面纯 markdown 写；不做居中/分页（那交给用户后续转 Word 处理）。

## 避免

- 不写 `<div align="center">`、`<br>`、`<center>` 等任何 HTML（会被当纯文本污染成品）。
- 不给封面加"（据《…》）"来源。
- 不在封面插图。
- 除那一行 `---` 外不加其它分隔线。
