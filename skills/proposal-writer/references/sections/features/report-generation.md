# 功能形态 · 报告生成类

系统生成并处理结构化报告的功能：报告生成、报告编辑、回写电子病历、一键引用等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：有报告/文档产出的系统（预问诊报告）。
- 不用：无报告产出的系统。

## 缺失信息优先提问顺序

1. 生成什么报告、含哪些内容
2. 报告后续如何流转（编辑、回写、引用）

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子：

- **预问诊报告生成**：依据对话生成结构化报告
- **报告编辑**：医生可修改
- **回写门诊电子病历**：报告写入 EMR
- **一键引用**：就诊时一键引用报告内容

## 写作要点

- 报告的**流转闭环**（生成→编辑→回写→引用）是核心价值，讲清。
- 回写病历涉及院内系统对接，只写素材确认的对接能力，别编 EMR 集成细节。
