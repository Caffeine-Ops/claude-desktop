# 功能形态 · 对话类

系统与用户的对话交互功能：多轮对话、对话引导、新会话、立即停止、点赞（踩）等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：对话式 AI 系统（导诊、预问诊）。
- 不用：无对话交互的系统。

## 缺失信息优先提问顺序

1. 对话有哪些能力（多轮、引导、打断、反馈等）
2. 对话的业务目标（导诊到科室？收集症状？）

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子：

- **对话引导**：主动引导用户逐步描述
- **多轮对话**：连续追问、上下文记忆
- **新会话 / 重新开始**：清空重来
- **立即停止**：中断生成
- **点赞（踩）**：对回复打分反馈

## 写作要点

- 每个对话能力一小节。
- 区分导诊场景与预问诊场景的对话（目标不同）。

## 本节常见呈现变体

- 逐条。
