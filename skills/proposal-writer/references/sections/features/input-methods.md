# 功能形态 · 输入类

用户向系统输入信息的功能：语音输入、文字输入、文字转语音等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：有多种输入交互的系统（导诊、预问诊）。
- 不用：无用户输入的纯展示系统。

## 缺失信息优先提问顺序

1. 支持哪些输入方式（素材里的输入功能清单）
2. 各输入方式的细节（如语音接什么设备、是否实时转写）

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条写各输入功能，常见叶子：

- **语音输入**：说话即可描述症状/需求
- **文字输入**：手动输入
- **文字转语音**：系统回复可语音播报

## 写作要点

- 每个输入方式一小节，含定义 + 说明 + 截图占位 + 来源。
- 只写素材支持的输入方式，别补造。

## 本节常见呈现变体

- 逐条（每种输入一小节）。

## 配图 / 结构图建议

- 每个输入功能配一张界面截图（满血）或占位（独立）。

## 正例 / 反例

- 正例：
  > **语音输入**
  > 患者可通过语音描述症状，系统实时转写为文字。
  > 〔建议插入：语音输入界面截图〕
  > （据《智能导诊产品说明》）
- 反例：把"语音、文字、图片、视频输入"一股脑列上，但素材只提到语音和文字——多出来的是编的。
