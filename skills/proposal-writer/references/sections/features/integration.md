# 功能形态 · 接入集成类

系统对接外部渠道/平台的功能：公众号、小程序、互联网医院、多院区、线下挂号患者支持等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：需接入多渠道/多平台的系统。
- 不用：单一封闭系统。

## 缺失信息优先提问顺序

1. 接入哪些渠道/平台
2. 各接入的范围（多院区？线上线下？）

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子：

- **微信公众号接入** / **小程序接入** / **互联网医院接入**
- **支持线下挂号患者**
- **多院区支持**

## 写作要点

- 接入清单要与"应用入口"（`sections/overview/entry-points.md`）一致。
- 只写素材确认的接入，别臆造渠道（如凭空加"抖音接入"）。

## 本节常见呈现变体

- 逐条或表格（渠道 | 接入方式 | 范围）。

## 配图 / 结构图建议

- 可配接入示意截图或一张 mermaid 接入拓扑。

## 正例 / 反例

- 正例：
  > **多院区支持**
  > 一套系统支持多院区独立配置，患者按所属院区进入对应服务。
  > （据《接入说明》）
- 反例：列了素材未提的"钉钉/飞书接入"——编造。
