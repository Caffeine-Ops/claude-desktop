# 功能形态 · 推荐类

系统给出推荐/建议的功能：科室推荐、医生推荐、对接挂号等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：有推荐/匹配能力的系统（导诊推荐科室）。
- 不用：无推荐逻辑的系统。

## 缺失信息优先提问顺序

1. 推荐什么（科室/医生/服务）
2. 推荐依据 + 推荐后能否直接行动（挂号）

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子：

- **科室推荐**：依据对话结果推荐匹配科室
- **医生推荐**：推荐对应出诊医生
- **对接挂号**：从推荐一键跳转挂号

## 写作要点

- 讲清推荐依据（凭什么推荐），但只写素材支持的机制，别夸大"精准 AI 推荐、准确率 99%"。
- 推荐到行动的闭环（推荐→挂号）是亮点，可强调。

## 本节常见呈现变体
