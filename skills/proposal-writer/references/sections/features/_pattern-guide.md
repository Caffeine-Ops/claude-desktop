# 功能详述 · 通用总则（先读）

"系统功能"是方案的**重点章、最详**。每个叶子功能按统一范式写，图文并茂。本卡是所有 `features/` 形态卡的共同底座。

## 叶子功能的统一写法（逐行）

每个叶子功能小节：

1. **小标题**（功能名，简短名词短语，用 markdown 标题层级，如 `#####`）
2. **一句定义**：这个功能是什么
3. **功能说明**：怎么用、有什么用（1–3 句，或按需展开）
4. **截图**：满血模式嵌产品截图 `![图说](<绝对路径>)`；独立模式写占位「〔建议插入：XX 界面截图〕」，
   **绝不编造图片路径**（见 `methodology/images-and-figures.md`）
5. **来源标注**：段末 `（据《文件名》）`（见 `methodology/grounding-and-citation.md`）

## 纪律

- 用户说"一条条介绍""逐条"的功能，**必须逐条**列举，别并成一段（见 `methodology/presentation-modes.md`）。
- 每个功能点**只写素材支持的**；素材没有的功能不编，缺细节标 `⚠️ 资料缺失`。
- 图文并茂：叶子功能凡所引文件有对应界面截图的应配一张，一节一图为宜（满血）；独立模式用占位。
- 功能层级与"系统功能架构"图一致，别矛盾。
