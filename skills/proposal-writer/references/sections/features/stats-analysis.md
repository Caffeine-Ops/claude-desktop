# 功能形态 · 统计分析类

系统的数据统计与分析功能：各类分析（潮汐/漏斗/画像）、报表、访问明细等。医疗方案里很重。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：有数据分析/运营看板的系统。
- 不用：无数据分析的系统。

## 缺失信息优先提问顺序

1. 有哪些分析维度/报表
2. 各分析给谁看、支撑什么决策

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子（医疗范例）：

- 智能导诊分析 / 预问诊分析 / 反馈分析 / 引用率分析 / 中断分析
- 潮汐分析 / 门诊转化漏斗分析 / 性别分析 / 年龄分析
- 全院/科室/医生 各级报表
- 访问明细 / 报告编辑明细

## 写作要点

- 分析类小节凡有图表截图的应配一张（满血）或占位（独立）。
- 分析维度**只写素材支持的**；别为凑数编造分析项。
- 涉及具体数值只在有素材出处时写，否则只描述"能分析什么"，不编数字。

## 本节常见呈现变体

- 逐条 + 图表截图/占位；报表清单可用表格。

## 配图 / 结构图建议

- 分析看板/图表截图（满血）或占位（独立）——统计类尤其该配图。

## 正例 / 反例

- 正例：
  > **门诊转化漏斗分析**
  > 统计从导诊到挂号、就诊的转化漏斗，定位流失环节。
  > 〔建议插入：转化漏斗看板截图〕
  > （据《数据分析说明》）
- 反例：编造"转化率 85%"等具体数字放进方案——无出处，删或标缺料。
