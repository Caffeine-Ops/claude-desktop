# 功能形态 · 后台管理类

系统的后台配置与管理功能：科室管理、提示词管理、对话轮次管理等。先读 `_pattern-guide.md`。

## 何时用 / 不用

- 用：有管理后台的系统。
- 不用：无后台配置的系统。

## 缺失信息优先提问顺序

1. 后台能管什么（配置项清单）
2. 谁来管、管理粒度

## 结构骨架（逐行）

按 `_pattern-guide.md` 逐条，常见叶子（医疗范例）：

- **门诊科室管理**：维护科室信息
- **提示词管理**：配置 AI 提示词
- **对话轮次管理**：设置对话轮数上限等

## 写作要点

- 后台功能通常**标准或精简篇幅**（见 `methodology/emphasis-and-depth.md`），别喧宾夺主。
- 只写素材支持的管理项，别补造配置能力。

## 本节常见呈现变体

- 逐条或表格（管理项 | 说明）。

## 配图 / 结构图建议

- 后台界面截图（满血）或占位（独立），一节一图为宜。

- 正例：
