# 系统总体目标（分受众视角）

按不同受益方分视角讲目标——这是建设方案的经典写法（尤其医疗：患者 / 医生 / 机构）。

## 何时用 / 不用

- 用：建设方案；有多类受益方时。
- 不用：单一受众的小方案，用一段总目标即可。

## 缺失信息优先提问顺序

1. 系统有哪几类受益方（医疗常见：患者、医生、医院）
2. 每类受益方从系统得到什么

## 结构骨架（逐行，每视角一小节）

以医疗为例（三视角，各一个三级小节）：

- **面向患者：提升就医体验** —— 减少排队/分诊困惑、报告更快等（逐条或一段）
- **面向医生：提高接诊效率** —— 预问诊报告减少问诊时间、回写病历等
- **面向机构：优化门诊运营管理** —— 分流、数据分析、资源配置、评级支撑

非医疗行业相应换视角（如：终端用户 / 一线员工 / 管理层）。

## 写作要点

- 每个视角的目标要**具体、可感知**，别写"提升效率"这种空话，要落到"怎么提升"。
- 目标须有素材依据；无据的目标标缺料。

## 本节常见呈现变体

- 默认：每视角一个三级小节 + 该视角下逐条目标。

## 配图 / 结构图建议

- 一般不配图；可用一张三视角价值示意 mermaid。

## 正例 / 反例

- 正例：「**面向患者**：候诊期间完成预问诊，就诊时医生已掌握病情概要，减少重复问诊。（据《方案说明》）」
- 反例：三视角写成一模一样的空话"提升效率、优化体验、赋能管理"——没区分度、没信息。
