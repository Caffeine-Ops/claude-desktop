# 系统应用入口

讲用户从哪些入口用到系统——公众号、小程序、互联网医院、Web、App 等。

## 何时用 / 不用

- 用：面向 C 端/多渠道的系统（医疗、政务办事、零售等）。
- 不用：纯内部系统单一入口时可省。

## 缺失信息优先提问顺序

1. 系统有哪些接入入口/渠道
2. 各入口面向谁、进入后能做什么

## 结构骨架（逐行）

- 逐条列各入口：入口名 + 面向的用户 + 主要用途
- 医疗常见：微信公众号 / 小程序 / 互联网医院 / 院内自助机

## 写作要点

- 入口列表要与"接入集成"功能（`sections/features/integration.md`）对齐、别矛盾。
- 只列素材支持的入口，不臆造渠道。

## 本节常见呈现变体

- 逐条或表格（入口 | 面向用户 | 用途）。

## 配图 / 结构图建议

- 可配一张入口示意截图（满血）或占位（独立）。

## 正例 / 反例

- 正例：「微信公众号：患者关注后进入导诊；小程序：预问诊；互联网医院：线上问诊接入。（据《接入说明》）」
- 反例：列了一个素材里根本没提的"支付宝小程序"——编造入口。
